# Create a Service Agreement: Step 1 – Summary and Design

## Overview

The **Step 1: Summary and Design** section allows users to configure the core details of a Service Agreement (SA), including:

- Agreement name
- Service Agreement template
- Business Unit
- Account ownership details
- PDF document design
- Agreement summary

## Things to Know

You can customize the PDF document generated for Service Agreements using **Template Manager**.

> **Recommendation:** Duplicate the default document template before making any changes. The original template is not editable, and using a copy helps preserve the default configuration while allowing customization.

## Where to Create a Service Agreement

You can create a Service Agreement from:

### Option 1
1. Open a **Customer Record** or **Location Record**.
2. In the **Service Agreements** section, click **Create Agreement**.

### Option 2
1. Open a **Customer Record** or **Location Record**.
2. Click **More**.
3. Select **Create Agreement**.

---

# Step 1: Summary and Design

Review and update the following fields:

## Agreement Name
Enter a name for the Service Agreement.

## Service Agreement Template
Select a Service Agreement template from the dropdown list.

## Business Unit
Select the appropriate Business Unit (BU).

## Account Manager (Optional)
Select the employee responsible for managing the agreement.

## Sold By (Optional)
Select the employee who sold the agreement.

## Tags (Optional)
Add tags to help categorize and organize Service Agreements.

### Example
Use a tag such as **Needs Review** to indicate that an agreement requires additional attention or follow-up.

Tags can be used to filter agreements in:

`Follow Up → Service Agreements`

## Document Template Design

Select the document template used to generate the Service Agreement PDF.

To preview the selected template:

1. Click **View Template**.
2. Review the template design in the new browser window.

## Agreement Summary (Optional)

Enter an internal summary of the Service Agreement.

This summary is intended for internal communication and is not included in customer-facing documents.
